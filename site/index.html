ANCService

Turn a $5 ESP32 into an Apple Notification Center Service bridge — your iPhone's notifications become events in Home Assistant. Follow the steps below; each takes about a minute, and you're done by the bottom.

1 Install the firmware

Plug the board into this computer with a USB cable, then click Install.

Your browser can't flash. Use desktop Chrome or Edge. Open this page over HTTPS to flash.

Normal install keeps saved WiFi and iPhone pairings. Use erase flash & install for a clean wipe (recovery, or re-pairing from scratch).

Trouble flashing? (“Failed to initialize”)
  1. Use desktop Chrome or Edge — not Safari, Firefox, or a phone.
  2. Use a data USB cable, and close anything else using the board (serial monitors, other browser tabs).
  3. If the port picker shows more than one device, choose the one matching your board’s USB chip — e.g. cu.SLAB_USBtoUART (CP2102).
  4. Still failing? Hold BOOT, click Install, pick the port, and keep holding BOOT through “Connecting…” until it starts writing — then release.

2 Set WiFi

During install, the wizard asks for your WiFi network and password and sends them to the device. That's it — no app needed.

Need to change WiFi later, or it didn't connect?

If the device can't join your network, it creates its own WiFi network called “ANCService Setup”. Join it from your phone or laptop and a setup page pops up automatically — pick your network, enter the password, and the device reboots onto your WiFi.

3 Add to Home Assistant

After flashing, click Add to Home Assistant in the installer dialog. Home Assistant discovers the device and adds it in a couple of clicks.

About encryption (optional)

The device joins Home Assistant without an encryption key — simplest to set up and fine for a trusted home network. For a per-device encrypted connection, use ESPHome's Take Control / adopt flow in the ESPHome dashboard, which regenerates the firmware with a unique key and updates it over the air.

4 Pair your iPhone

Apple no longer lists custom accessories in Settings → Bluetooth, so you pair once using the free nRF Connect for Mobile app. After that your iPhone reconnects on its own.

  1. Install nRF Connect for Mobile (free) and make sure Bluetooth is on.
  2. Open nRF Connect → SCANNER → tap SCAN.
  3. Find your device — it appears as ancservice- followed by a short code (e.g. ancservice-a1b2c3) — and tap CONNECT.
  4. Tap Pair on the iPhone dialog that appears.
  5. Tap Allow on the “receive notifications” dialog — this grants access.
  6. Close nRF Connect. Done — the bond is stored by iOS and reconnects automatically.
Why do I need a separate app?

Around iOS 16, Apple stopped showing custom Bluetooth accessories in Settings → Bluetooth. nRF Connect just opens a connection so iOS shows its standard pairing dialog — the same one you'd normally tap in Settings. You only need it for this one-time pairing; it's never needed again.

Pairing didn't take?

In iOS Settings → Bluetooth, tap Forget This Device if ANCService is listed, then repeat the steps above from your scan.

5 Use it in Home Assistant

Your iPhone's notifications now arrive in Home Assistant as events you can automate on:

EventFires when
esphome.ancs_incoming_callan incoming call arrives
esphome.ancs_notificationany other notification arrives
esphome.ancs_call_endeda call is answered or declined

Example — flash a light when a call comes in (Settings → Automations → Edit as YAML):

trigger:
  - platform: event
    event_type: esphome.ancs_incoming_call
action:
  - service: light.turn_on
    target:
      entity_id: light.YOUR_LIGHT
    data:
      flash: long
More automation examples

Stop flashing when the call ends:

trigger:
  - platform: event
    event_type: esphome.ancs_call_ended
action:
  - service: light.turn_off
    target:
      entity_id: light.YOUR_LIGHT

Announce the caller out loud:

trigger:
  - platform: event
    event_type: esphome.ancs_incoming_call
action:
  - service: tts.cloud_say
    data:
      entity_id: media_player.YOUR_SPEAKER
      message: "Call from {{ trigger.event.data.caller }}"

Only react to a specific app (e.g. Messages) — match on app_id:

trigger:
  - platform: event
    event_type: esphome.ancs_notification
    event_data:
      app_id: com.apple.MobileSMS
